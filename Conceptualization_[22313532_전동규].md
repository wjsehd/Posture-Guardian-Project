### 1. Business purpose

**1) Project background**

[현대인의 고질병: VDT 증후군과 인지 부조화]
<br>
전공 과제를 수행하거나 장시간 코딩에 몰입하다 보면, 사용자는 자신도 모르게 목이 앞으로 빠지고 어깨가 말리는 이른바 '거북목(Forward Head Posture)' 자세를 취하게 된다. 이는 현대 사회의 전형적인 VDT(Visual Display Terminal) 증후군의 일종으로, 경추 하부의 부하를 급격히 증가시켜 목과 어깨의 만성 통증, 나아가 디스크 질환을 유발한다. 특히 심리학적 관점에서 인간은 고도의 몰입(Flow) 상태에 진입할 때 신체적 감각에 대한 인지 능력이 저하되는데, 이로 인해 자세가 완전히 무너진 뒤 통증이라는 '사후 신호'가 오기 전까지는 스스로 자세를 교정하기가 사실상 불가능하다.

[기존 솔루션의 한계: 물리적/심리적 진입장벽]
<br>
현재 시장에는 자세 교정 밴드나 고가의 기능성 의자 등 다양한 물리적 솔루션이 존재한다. 그러나 이러한 방식은 매번 장비를 착용해야 하는 번거로움과 경제적 부담이라는 명확한 한계가 있다. 또한, 단순히 정해진 시간마다 알람을 울려주는 스트레칭 앱들은 사용자의 실제 자세와 동기화되지 않으므로, 집중력을 불필요하게 분산시킬 뿐 실질적인 행동 변화를 이끌어내지 못한다. 즉, 사용자의 현재 상태를 실시간으로 모니터링하면서도 업무 흐름을 방해하지 않는 '비접촉식 시스템'이 절실한 시점이다.

[기술적 대안: AI 포즈 추정 기반의 실시간 능동 피드백]
<br>
따라서 본 프로젝트는 별도의 하드웨어 구매 없이, 노트북이나 모니터에 기본 탑재된 '웹캠(Webcam)'을 지능형 센서로 활용하는 소프트웨어적 해결책을 제안한다. 이를 위해 컴퓨터 비전의 핵심 기술인 '포즈 추정(Pose Estimation)' 알고리즘을 도입한다.

포즈 추정이란 딥러닝 기반의 인공 신경망을 통해 영상 속 인체의 주요 관절 위치를 탐지하고 좌표화하는 기술이다. 본 시스템은 사용자의 눈, 귀, 어깨의 수직/수평 좌표를 실시간으로 연산하여 목의 기울기 각도를 정밀하게 측정한다. 특히 MediaPipe와 같은 경량화된 AI 모델을 활용함으로써, 시스템 리소스 점유율을 최소화하여 코딩이나 고사양 작업 중에도 PC 성능 저하 없이 상시 구동이 가능하다. 결과적으로 나쁜 습관이 고착화되는 '찰나의 순간'에만 시각적·청각적 피드백을 제공함으로써, 사용자가 스스로 바른 자세를 학습할 수 있도록 돕는 지능형 헬스케어 보조 도구를 기획하게 되었다.

**2) Goal**

이 프로젝트의 핵심 목표는 사용자가 PC를 사용하는 동안 올바른 자세를 유지할 수 있도록 보조하는 자동화된 피드백 시스템을 개발하는 것이다.

본 프로젝트의 궁극적인 목표는 사용자가 인지하지 못하는 사이 무너지는 자세를 실시간으로 교정하여, 장기적으로 올바른 PC 사용 습관을 형성하도록 돕는 '지능형 자세 관리 솔루션'을 구축하는 것이다. 이를 위해 다음과 같은 세부 목표를 설정한다.

* **컴퓨터 비전 기반의 정밀한 실시간 자세 분석**
단순한 움직임 감지를 넘어, 웹캠 프레임에서 눈(Eyes), 귀(Ears), 어깨(Shoulders)의 다차원 좌표(Keypoints)를 0.1초 단위로 정밀하게 추출한다. 이를 통해 거북목의 핵심 지표인 '목의 전방 기울기'를 수치화하여 분석의 신뢰도를 확보한다.

* **사용자 몰입을 방해하지 않는 지능형 피드백 시스템**
사용자의 작업 흐름(Workflow)을 깨지 않는 것이 핵심이다. 단순한 1회성 알림이 아니라, 불량 자세가 특정 시간(3초 등) 이상 지속되는 '패턴'을 감지했을 때만 시각적 팝업이나 비프음을 발생시킨다. 또한 사용자의 신체 조건에 맞춘 영점 설정(Calibration) 기능을 통해 오작동을 최소화한다.

* **데이터 기반의 장기적 자세 개선 가이드 제공**
일회성 교정에 그치지 않고, 매일 발생하는 자세 데이터를 데이터베이스화한다. 주간/월간 단위의 자세 건강 리포트를 시각화(그래프)하여 제공함으로써, 사용자가 본인의 자세 변화를 정량적으로 확인하고 스스로 나쁜 습관을 고쳐나갈 수 있는 동기를 부여한다.

* **시스템 최적화 및 안정성 확보**
백그라운드에서 상시 구동되는 프로그램 특성상, CPU 및 메모리 점유율을 최소화하는 경량화 알고리즘을 구현한다. 이를 통해 개발 환경(IDE)이나 고사양 작업 중에도 시스템 성능 저하 없이 안정적인 서비스를 제공하는 것을 목표로 한다.

<br>

**3) Target Market**

* 컴퓨터 앞에서 오랜 시간 코딩을 하거나 문서 작업을 해야 하는 개발자 및 사무직 직장인.
* 비대면 인터넷 강의를 시청하거나 PC로 과제를 많이 수행하는 대학생 및 수험생.
* 별도의 교정 장비 없이, 소프트웨어 설치만으로 일상 속에서 거북목 증후군을 예방하고 싶은 일반 사용자.

### 2. System context diagram

<div align="center">

<br>

이 시스템은 사용자의 PC에서 카메라 영상을 실시간으로 처리하는 클라이언트 프로그램(System)과, 사용자의 장기적인 통계 기록을 보관하는 백엔드 데이터베이스 서버(DB Server)로 나뉘어 동작한다.

**[ System Context Diagram 구성 요소 ]**

* **User (Actor):** 프로그램을 실행하고 모니터 앞에 앉아있는 실제 사용자. 최초 실행 시 자신의 바른 자세를 영점(Calibration)으로 등록하고, 시스템에 웹캠 사용 권한을 제공한다.
* **System (Posture Guardian):** PC 백그라운드에서 동작하는 핵심 소프트웨어. 카메라 프레임을 받아와 자세 불량 여부를 지속적으로 연산하며, 이벤트가 발생할 때마다 화면에 경고 알림을 출력한다.
* **DB Server (Actor):** 클라이언트가 보내는 통계 데이터를 저장하는 원격 서버. 사용자가 다른 기기에서 접속하더라도 동일한 누적 통계와 알림 환경 설정을 유지할 수 있도록 데이터를 관리한다.

**[ 상호작용 세부 명세 (Interactions) ]**

**▶ User ↔ System 간 상호작용**
* **Provide Video Frame (입력):** User는 System 측으로 초당 일정 프레임의 실시간 웹캠 이미지를 지속적으로 제공한다.
* **Calibration & Setting (설정):** User는 허리를 편 바른 자세를 System에 초기 기준으로 등록하고, 알림을 받을 각도와 방식을 설정한다.
* **Request Dashboard (조회 요청):** User는 자신의 주간 자세 점수를 확인하기 위해 통계 리포트 화면을 연다.
* **Push Notification (경고 출력):** System은 연산 결과 자세 불량이 감지되면 User의 모니터 화면에 강제로 시스템 알림 창을 띄운다.
* **Display Statistics (통계 출력):** System은 누적된 데이터를 기반으로 User에게 시각적인 그래프 리포트를 보여준다.

**▶ System ↔ DB Server 간 상호작용**
* **Send Log Data (데이터 백업):** System은 '14시 30분 경고 1회 발생'과 같은 텍스트 기반의 메타데이터만 DB Server로 전송한다. (사생활 보호를 위해 웹캠 영상 자체는 절대 서버로 전송하지 않고 연산 즉시 메모리에서 폐기한다.)
* **Fetch User Data (동기화):** System은 프로그램이 켜질 때 DB Server로부터 User가 예전에 저장해둔 설정값과 과거 통계 기록을 불러온다.
* **Acknowledge (수신 확인):** DB Server는 System이 보낸 로그 데이터가 정상적으로 저장되었음을 알리는 응답 신호를 보낸다.

### 5. Problem statement

> **Overview**
> '거북목 교정'은 사용자가 스스로 인지하지 못하는 순간에도 자세를 파악하고, 시스템이 실시간으로 교정을 보조하는 것이 목적이다. 이 시스템은 다음과 같은 목적을 달성해야 한다.
> * 실시간 골격 좌표 추출 및 연산
> * 정상 범위 기준점(Calibration) 설정 및 비교
> * 임계값 초과 시 즉각적인 경고 피드백 제공
> * 자세 경고 로그의 데이터베이스화 및 통계 제공

<br>

**1) Problem #1**
정확한 영점 기준과 판단이 있어야 한다. 사용자마다 신체 조건과 카메라 환경이 다르기 때문에 기준이 고정되어 있다면 잘못된 경고를 발생시키게 되고 이 어플리케이션의 목적을 상실하게 된다. 또한, 잦은 오작동은 사용자의 작업 몰입도와 PC 사용에 치명적인 영향을 줄 수 있다.

**2) Problem #2**
영상 처리 과정에서 시스템 리소스 점유율이 최소화되어야 한다. 실시간 영상 분석으로 인해 PC의 CPU 및 메모리 점유율이 과도하게 높아진다면, 사용자는 원래 목적이었던 문서 작업이나 코딩 등의 주 기능으로 접근이 불가능하다.

**3) Problem #3**
사용자의 웹캠 영상을 전산화하여 사용하면 개인의 private information 문제가 발생할 수 있다. 연산을 위해 캡처되는 영상에는 사용자의 얼굴과 사생활 공간이 노출되는 등 여러 경우를 고려해야 한다. 또한, 영상 데이터가 외부 서버로 전송되어 전산이 해킹된다면 치명적인 개인 정보 유출로 이어질 수 있다.

### 6. Glossary

| Terms | Description |
| :--- | :--- |
| **사용자** | 웹캠을 통해 자세를 모니터링 받고 교정 알림을 받는 대상 |
| **DB 서버** | 시스템에서 발생한 경고 로그 및 통계 데이터를 저장하고 관리하는 원격 서버 |
| **영점 설정** | 정확한 판별을 위해 사용자의 바른 자세를 최초 기준으로 시스템에 등록하는 행위 |
| **관절 포인트** | 영상 데이터 분석을 통해 추출해 내는 눈, 귀, 어깨 등의 신체 좌표 지점 |
| **임계값** | 거북목으로 판정하여 경고 알림을 발생시키는 기준이 되는 목의 기울기 각도 및 지속 시간 |
| **데이터 테이블** | 사용자의 영점 설정값 및 환경 설정 정보가 저장된 테이블 |
| **경고 로그 테이블** | 자세 불량이 감지되어 알림이 발생한 시각, 유지 시간 등의 현황 정보가 저장된 테이블 |
