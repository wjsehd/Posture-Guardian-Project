### 1. Business purpose

**1) Project background**
전공 과제를 하거나 코딩을 위해 장시간 모니터 앞에 앉아 있다 보면, 자연스럽게 목이 앞으로 빠지고 어깨가 말리는 이른바 '거북목' 자세가 되기 쉽다. 처음에는 의식해서 허리를 펴고 앉더라도, 화면에 집중하는 순간 자신도 모르게 자세가 무너지게 된다. 문제는 작업 도중에는 스스로 자세가 틀어졌다는 것을 인지하기가 거의 불가능하며, 보통 목이나 어깨에 통증이 느껴지고 나서야 문제를 깨닫는다는 점이다.

시중에는 자세 교정 밴드나 기능성 의자 같은 물리적인 제품들이 많지만, 매번 챙겨서 착용해야 하는 번거로움이 있고 가격적인 부담도 있다. 스마트폰이나 PC의 알람을 맞춰두고 주기적으로 스트레칭을 하도록 유도하는 앱들도 존재하지만, 사용자의 실제 자세와는 무관하게 정해진 시간에만 울리기 때문에 실질적인 교정 효과는 미미하다.

따라서 따로 돈을 들여 장비를 살 필요 없이, 노트북이나 모니터에 기본적으로 달려있는 '웹캠'을 활용해 내 자세를 봐주는 프로그램이 있으면 어떨까 생각했다. 카메라 영상으로 사용자의 상체 골격을 실시간으로 추적하다가, 자세가 흐트러지는 바로 그 순간에 모니터 화면에 경고를 띄워준다면 나쁜 습관을 고치는 데 훨씬 효과적일 것이라 판단하여 본 시스템을 기획하게 되었다.

**2) Goal**
이 프로젝트의 핵심 목표는 사용자가 PC를 사용하는 동안 올바른 자세를 유지할 수 있도록 보조하는 자동화된 피드백 시스템을 개발하는 것이다.
* **실시간 골격 인식:** 사용자의 웹캠에서 들어오는 영상 프레임을 분석하여 눈, 귀, 어깨의 좌표를 추출하고 현재 목의 기울기를 계산한다.
* **맞춤형 경고 피드백:** 사용자의 목 각도가 사전에 설정해둔 정상 범위를 벗어난 상태로 3초 이상 유지되면, 화면 우측 하단에 알림 팝업을 띄우거나 비프음을 발생시켜 즉각적으로 자세를 고쳐 앉도록 유도한다.
* **자세 데이터 로깅 및 시각화:** 하루 동안 발생한 경고 횟수와 나쁜 자세를 유지한 시간 등을 데이터베이스에 기록하여, 주간 단위로 자신의 자세가 얼마나 개선되었는지 확인할 수 있는 통계 화면을 제공한다.

**3) Target Market**
* 컴퓨터 앞에서 오랜 시간 코딩을 하거나 문서 작업을 해야 하는 개발자 및 사무직 직장인.
* 비대면 인터넷 강의를 시청하거나 PC로 과제를 많이 수행하는 대학생 및 수험생.
* 별도의 교정 장비 없이, 소프트웨어 설치만으로 일상 속에서 거북목 증후군을 예방하고 싶은 일반 사용자.


### 2. System context diagram

이 시스템은 사용자의 PC에서 카메라 영상을 실시간으로 처리하는 클라이언트 프로그램(System)과, 사용자의 장기적인 통계 기록을 보관하는 백엔드 데이터베이스 서버(DB Server)로 나뉘어 동작한다.

**[ System Context Diagram 구성 요소 ]**

* **User (Actor):** 프로그램을 실행하고 모니터 앞에 앉아있는 실제 사용자. 최초 실행 시 자신의 바른 자세를 영점(Calibration)으로 등록하고, 시스템에 웹캠 사용 권한을 제공한다.
* **System (Posture Guardian):** PC 백그라운드에서 동작하는 핵심 소프트웨어. 카메라 프레임을 받아와 자세 불량 여부를 지속적으로 연산하며, 이벤트가 발생할 때마다 화면에 경고 알림을 출력한다.
* **DB Server (Actor):** 클라이언트가 보내는 통계 데이터를 저장하는 원격 서버. 사용자가 다른 기기에서 접속하더라도 동일한 누적 통계와 알림 환경 설정을 유지할 수 있도록 데이터를 관리한다.

**[ 상호작용 세부 명세 (Interactions) ]**

**▶ User ↔ System 간 상호작용**
* **Provide Video Frame (입력):** User는 System 측으로 초당 일정 프레임의 실시간 웹캠 이미지를 지속적으로 제공한다.
* **Calibration & Setting (설정):** User는 허리를 편 바른 자세를 System에 초기 기준으로 등록하고, 알림을 받을 각도와 방식을 설정한다.
* **Request Dashboard (조회 요청):** User는 자신의 주간 자세 점수를 확인하기 위해 통계 리포트 화면을 연다.
* **Push Notification (경고 출력):** System은 연산 결과 자세 불량이 감지되면 User의 모니터 화면에 강제로 시스템 알림 창을 띄운다.
* **Display Statistics (통계 출력):** System은 누적된 데이터를 기반으로 User에게 시각적인 그래프 리포트를 보여준다.

**▶ System ↔ DB Server 간 상호작용**
* **Send Log Data (데이터 백업):** System은 '14시 30분 경고 1회 발생'과 같은 텍스트 기반의 메타데이터만 DB Server로 전송한다. (사생활 보호를 위해 웹캠 영상 자체는 절대 서버로 전송하지 않고 연산 즉시 메모리에서 폐기한다.)
* **Fetch User Data (동기화):** System은 프로그램이 켜질 때 DB Server로부터 User가 예전에 저장해둔 설정값과 과거 통계 기록을 불러온다.
* **Acknowledge (수신 확인):** DB Server는 System이 보낸 로그 데이터가 정상적으로 저장되었음을 알리는 응답 신호를 보낸다.
