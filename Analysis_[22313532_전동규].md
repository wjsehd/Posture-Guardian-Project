## 1. Introduction

### 1) Executive Summary
현대 사회에서 전공 과제나 코딩 등 장시간 PC 작업에 몰입하는 사용자들은 자신도 모르게 목이 앞으로 빠지는 '거북목(Forward Head Posture)' 자세를 취하게 된다. 이는 VDT(Visual Display Terminal) 증후군을 유발하여 목과 어깨의 만성 통증으로 이어진다. 기존의 자세 교정 밴드나 고가의 기능성 의자는 물리적/경제적 진입장벽이 존재하며, 단순한 시간 알림 앱은 사용자의 실제 자세와 동기화되지 않아 실질적인 교정 효과가 미미하다.
따라서 본 프로젝트인 "Posture Guardian"은 별도의 하드웨어 추가 없이 노트북이나 모니터에 내장된 웹캠(Webcam)을 지능형 센서로 활용하는 소프트웨어 기반의 헬스케어 보조 시스템이다. 컴퓨터 비전의 포즈 추정(Pose Estimation) 알고리즘을 이용해 사용자의 자세를 실시간으로 분석하고, 잘못된 자세가 고착화되는 순간에만 개입하여 올바른 PC 사용 습관을 형성하도록 돕는다.

### 2) Business Goals
Posture Guardian 시스템의 핵심 비즈니스 목표는 '사용자의 작업 몰입도를 방해하지 않으면서도 확실한 자세 교정 효과를 제공'하는 것이다. 
이를 위해 일회성 알람이 아닌, 불량 자세가 특정 시간 이상 지속되는 패턴을 시스템이 인지했을 때만 비침투적(Non-intrusive)인 방식으로 피드백을 제공한다. 또한, 매일의 자세 데이터를 수집 및 수치화하여 사용자에게 시각적인 건강 리포트를 제공함으로써, 사용자 스스로 자신의 문제를 인지하고 자발적으로 바른 자세를 유지하려는 장기적인 동기부여를 이끌어낸다.

### 3) Technical Goals
본 시스템은 PC 백그라운드에서 상시 구동되는 클라이언트 프로그램과, 통계 데이터를 관리하는 원격 DB 서버로 구성된다.
핵심 기술 목표는 사용자 PC의 리소스(CPU, 메모리) 점유율을 최소화하는 것이다. 이를 위해 MediaPipe와 같은 경량화된 딥러닝 기반 포즈 추정 모델을 도입하여, 무거운 개발 환경(IDE)이나 고사양 작업 중에도 성능 저하 없이 눈, 귀, 어깨의 다차원 좌표를 정밀하게 추출한다.
또한, 사생활 침해 및 보안 문제를 원천 차단하기 위해 캡처된 웹캠 영상은 좌표 연산 즉시 클라이언트 메모리에서 폐기하며, DB 서버로는 알림 발생 시각 및 빈도 등의 텍스트 기반 메타데이터(Meta Data)만 암호화하여 전송하는 안전한 데이터 처리 아키텍처를 구현한다.

### 4) Non-Functional Requirements
시스템이 원활하게 동작하고 사용자에게 신뢰성을 제공하기 위해 달성해야 하는 비기능적 요구사항은 다음과 같다.

* **Performance (성능):**
  * 실시간 자세 분석의 지연을 막기 위해 영상 프레임 처리 속도는 최소 15 FPS 이상을 유지해야 한다.
  * 백그라운드 AI 연산 중에도 사용자의 주 작업(IDE 컴파일, 게임 등)에 영향을 주지 않도록 CPU 점유율을 최적화해야 한다.
* **Security & Privacy (보안 및 사생활 보호):**
  * 웹캠을 통해 수집된 사용자의 원본 영상 프레임은 좌표 추출 연산 즉시 메모리에서 영구 삭제되어야 하며, 어떠한 경우에도 외부 서버로 전송되거나 로컬 디스크에 저장되어서는 안 된다.
  * 외부 DB 서버와의 모든 통신(로그인, 통계 동기화)은 HTTPS 프로토콜을 통해 암호화되어야 한다.
* **Usability (사용성):**
  * 자세 경고 알림은 사용자의 작업 몰입도를 방해하지 않도록 화면의 10% 이하 면적을 차지하는 비침투적(Non-intrusive) 토스트 팝업 형태로 제공되어야 한다.
  * 시스템 최초 실행 시, 사용자가 매뉴얼 없이도 1분 이내에 영점 설정(Calibration)을 완료할 수 있도록 직관적인 UI 가이드를 제공해야 한다.
* **Availability (가용성):**
  * 오프라인(인터넷 미연결) 상태에서도 실시간 모니터링과 경고 알림, 로컬 로그 저장은 100% 정상 작동해야 한다.

* **Naming Convention**: 각 Use Case는 시스템이 수행하는 동작을 명확하게 전달하기 위해 동사(Verb)로 시작하도록 Naming하였다.
* **Actors**: 본 프로젝트에서는 User(사용자), System(본 프로그램), 그리고 외부 협력 객체인 DB Server를 Actor로 설정하여 각 주체 간의 상호작용을 나타내었다.
* **Relationships**: 각 기능의 연관성에 따라 include(포함) 관계와 extend(확장) 관계를 적용하여 시스템의 논리적 흐름을 상세화하였다.

**[ 주요 관계 설명 ]**

* **include (포함 관계)**: 특정 기능을 수행하기 위해 반드시 선행되어야 하는 필수 과정을 정의하였다.
  * `View Report`와 `Sync Data` 기능 수행 시 사용자 인증을 위해 `Manage Account` 과정이 반드시 포함된다.
  * `Analyze Posture` 및 `Camera Calibration`을 수행하기 전, 영상 소스 확보를 위한 `Select Camera`가 필수적으로 실행된다.

* **extend (확장 관계)**: 기본 유스케이스 수행 중 특정 조건이 만족될 때만 선택적으로 발생하는 기능을 정의하였다.
  * `Analyze Posture` 과정 중 사용자의 자세가 설정한 임계값을 벗어난다는 조건이 충족될 때만 `Send Alert` 기능이 확장되어 실행된다.

### 3.1. Domain Class Description

1) **UserAccount** : 시스템을 사용하는 User Actor가 갖는 Attribute와 Operation을 정의하는 클래스이다. (예를 들어 ID, Password, 인증 토큰 등) 시스템이 처음 시작할 때 회원의 로그인 정보를 확인하고 인증을 처리한다.

2) **CameraManager** : 다수의 카메라 중 자세 분석에 사용할 비디오 입력 장치를 선택하고 제어하는 기능을 가진 클래스이다. 또한 사용자의 바른 자세(영점) 골격 좌표를 최초 기준으로 추출하여 등록하는 역할(Calibration)을 수행한다.

3) **SettingManager** : 사용자가 경고를 받기 위해 지정한 목의 기울기 각도 및 지속 시간(Threshold)과, 프라이버시 보호를 위한 모니터링 기능의 ON/OFF 제어 상태를 저장하고 관리하는 클래스이다.

4) **PostureAnalyzer** : 시스템의 핵심 비즈니스 로직을 담당하는 클래스이다. 웹캠으로부터 전달받은 영상 프레임에서 사용자의 관절 포인트를 실시간으로 추출하고, 영점 기준과 비교하여 자세 불량 여부를 지속적으로 연산한다.

5) **AlertManager** : `PostureAnalyzer`의 연산 결과가 사용자가 설정한 임계값을 초과하여 유지될 때, 화면에 경고 팝업 알림을 발생시키는 클래스이다. 알림 발생 시 해당 이벤트 시간과 각도 등을 로컬 로그 데이터로 기록한다.

6) **DataSync** : `AlertManager`가 생성한 로컬 경고 로그 데이터를 모아, 정해진 주기마다 외부 서버로 전송하고 백업(동기화)하는 기능을 담당하는 클래스이다.

7) **ReportViewer** : 사용자가 자신의 건강 점수나 자세 개선 추이를 확인하고자 할 때, 서버로부터 누적 통계 데이터를 가져와 대시보드 형태의 시각적 그래프로 화면에 출력하는 클래스이다.

8) **DBServer** : 일종의 "Server" 역할을 담당하는 외부 시스템 클래스이다. 시스템에 가입한 회원들의 계정 정보, 사용자별 환경 설정, 장기 누적된 자세 경고 통계 로그 등 여러 데이터를 안전하게 보관하고 제공한다.

### 3.2. Detailed Class Specification

도메인 모델에서 정의된 각 클래스의 세부 속성(Attributes)과 주요 메서드(Operations)를 명세한다. 이 명세는 향후 시스템 구현 시 클래스 설계의 기초 데이터로 활용된다.

| Class Name | Attributes (속성) | Operations (주요 메서드) |
| :--- | :--- | :--- |
| **UserAccount** | userID: String<br>password: String<br>authToken: String | login(): boolean<br>register(): boolean<br>logout(): void |
| **CameraManager** | cameraList: List<br>selectedCamera: Device<br>baseCoords: Point[] | getAvailableCameras(): Device[]<br>selectCamera(id): void<br>calibrate(): void |
| **SettingManager** | thresholdAngle: int<br>thresholdTime: int<br>isMonitoringOn: boolean | updateThresholds(angle, time): void<br>toggleMonitoring(): void |

## 4. User Interface prototype

본 시스템의 사용자 인터페이스(UI)는 장시간 PC 화면을 주시해야 하는 타겟 사용자(대학생, 개발자, 직장인 등)의 특성을 고려하여 설계되었다. 눈의 피로도를 최소화하고 화면의 눈부심을 방지하기 위해 전체적인 디자인 시스템을 **다크 모드(Dark Theme)** 기반으로 구축하였다. 또한, 복잡한 화면 이동(Depth)을 배제하고 화면 좌측에 글로벌 네비게이션 바(GNB)를 고정 배치하여, 사용자가 어느 화면에서든 4개의 핵심 기능(대시보드, 영점 설정, 통계 리포트, 환경 설정)으로 즉각적인 접근이 가능하도록 직관성을 극대화하였다.

### 4.1. 메인 대시보드 (Main Dashboard)

  * **실시간 자세 피드백 영역:** 시스템이 분석한 사용자의 현재 목 기울기 편차(예: $26.7^\circ$)를 큰 타이포그래피로 실시간 렌더링한다. 임계값을 초과한 불안정한 자세일 경우 수치와 경고 아이콘이 붉은색으로 강조되어 즉각적인 시각적 인지를 돕는다. (Use Case #6)
  * **미니 대시보드 (점수 및 설정 요약):** 통계나 설정 메뉴로 이동하지 않고도, 하단의 미니 라인 차트를 통해 최근 7일간의 건강 점수 추이를 가볍게 훑어볼 수 있으며, 현재 적용된 임계값(각도 및 허용 시간)을 요약 뷰로 제공한다.
  * **비침투적 경고 팝업 (Toast Alert):** 자세가 설정된 임계값을 초과하여 일정 시간 유지될 경우, 사용자의 주 작업 화면(IDE, 문서 등)을 강제로 가리거나 방해하지 않도록 우측 하단에 부드럽게 나타나는 토스트 팝업 형태로 경고를 발생시킨다. (Use Case #7)

### 4.3. 통계 리포트 (Statistics Report)
단순한 경고의 나열을 넘어, 축적된 자세 로그 데이터를 가공하여 시각화된 건강 차트로 제공하는 화면이다. 사용자가 자신의 상태를 객관적으로 인지하고 자발적인 습관 개선을 하도록 동기를 부여하는 비즈니스 목표를 달성하는 핵심 UI이다.

## 5. Glossary
본 문서에서 사용된 주요 도메인 개념 및 시스템 설계 용어의 정의는 다음과 같다. 개념화 단계에서 정의된 핵심 용어에 더하여, 분석 및 설계 과정에서 구체화된 기술적/기능적 용어를 포함한다.

| Terms | Description |
| :--- | :--- |
| **사용자 (User)** | 웹캠을 통해 자세를 모니터링 받고 교정 알림을 받는 대상 |
| **DB 서버 (DB Server)** | 시스템에서 발생한 경고 로그 및 통계 데이터를 저장하고 관리하는 원격 서버 |
| **거북목 증후군 (Forward Head Posture)** | 머리가 몸의 수직 중심축보다 앞으로 빠져나와 목과 어깨 등 척추 주변 근육에 과도한 하중이 가해지는 만성적 자세 불량 상태 |
| **VDT 증후군 (Visual Display Terminal Syndrome)** | PC 모니터 등 영상 기기를 장시간 집중해서 사용함에 따라 발생하는 목/어깨 통증 등의 복합 질환군 |
| **포즈 추정 (Pose Estimation)** | 컴퓨터 비전 알고리즘을 이용해 실시간 영상 속 인물의 주요 관절 포인트 위치를 추적하고 공간적 구조를 분석하는 기술 |
| **관절 포인트 (Landmarks)** | 영상 데이터 분석을 통해 추출해 내는 눈, 귀, 어깨 등의 신체 좌표 지점 |
| **비디오 입력 장치 (Video Input Device)** | 사용자 PC에 연결되어 실시간 영상 스트림을 시스템에 제공하는 다수의 카메라(웹캠) 중 선택된 타겟 장치 |
| **영점 설정 (Calibration)** | 정확한 판별을 위해 사용자의 바른 자세를 최초 기준으로 시스템에 등록하는 행위 |
| **임계값 (Threshold)** | 거북목으로 판정하여 경고 알림을 발생시키는 기준이 되는 목의 기울기 각도 및 지속 시간 |
| **비침투적 피드백 (Non-intrusive Feedback)** | 사용자의 주 작업(코딩, 문서 작성 등) 흐름을 강제로 끊지 않고 팝업 등을 통해 간접적으로 개입하는 알림 방식 |
| **모니터링 제어 (Control Monitoring)** | 사용자가 사생활 보호 및 자원 관리를 위해 언제든지 AI 영상 분석 스레드를 명시적으로 켜거나 끌 수 있는 기능 |
| **데이터 테이블 (Data Table)** | 사용자의 영점 설정값 및 환경 설정 정보가 저장된 테이블 |
| **경고 로그 테이블 (Alert Log Table)** | 자세 불량이 감지되어 알림이 발생한 시각, 유지 시간 등의 현황 정보가 저장된 테이블 |
| **동기화 (Sync)** | 다른 기기나 환경에서도 DB 서버와 통신하여 사용자의 누적 통계 데이터 및 설정값을 동일하게 맞추는 작업 |
| **인증 토큰 (Auth Token)** | 계정 로그인 성공 시 DB 서버로부터 발급받아 시스템과 서버 간의 안전한 통신(세션 유지)에 사용하는 고유 식별자 |
